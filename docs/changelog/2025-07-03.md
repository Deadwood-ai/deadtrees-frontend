# Phenology Visualization Feature

**Date:** 2025-01-02

## Added

- **Phenology Bar Component** - Interactive timeline showing vegetation activity across the year with data-driven grey-to-green gradient
- **Environmental Context Section** - New grouped section in Dataset Details combining biome, acquisition date, vegetation activity tag, and growing season visualization
- **Vegetation Activity Indicator** - Color-coded tags (red/orange/green) showing immediate activity level for acquisition date
- **Audit Integration** - Added phenology assessment to dataset audit workflow with proper spacing

## Features

- Hover interactions with precise day indicators and detailed tooltips
- Blue acquisition date markers showing when data was captured
- Month labels and responsive design
- MODIS phenology data integration with 5-minute caching
- Consistent Ant Design color scheme

## Technical Details

- **Data Source**: MODIS Phenology curves stored in `v2_metadata` table as JSONB
- **Data Structure**: 366 daily values (0-255 scale) representing vegetation activity
- **Activity Thresholds**:
  - Low Activity (🟤 Red): 0-85 values - dormant/winter season
  - Moderate Activity (🟡 Orange): 86-170 values - transitional periods
  - High Activity (🟢 Green): 171-255 values - peak growing season
- **Color Gradient**: Linear interpolation from #F5F5F5 (light grey) to #388E3C (dark green)
- **Positioning**: Day-of-year calculations using 366-day timeline for leap year compatibility
- **Month Positioning**: Labels positioned at month boundaries (day 1, 32, 60, etc.)

## Impact

Users can now immediately assess whether datasets were captured during optimal growing season for dead tree detection, improving data quality evaluation and analysis planning.
